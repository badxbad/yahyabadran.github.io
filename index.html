The BtLibrary

This Library will be available with Version 4.0 as an extra choice for those who want to have more control over the Android Bluetooth API.

Connecting by Name or Mac Address

We need the TechTweaking.Bluetooth.BluetoothDevice which will be the remote device that we connect to.

BluetoothDevice device = new BluetoothDevice();

Identify the device either by its mac address or by its name, so the BtLibrary can find it and connect to it:

device.Name = "HC-05";

or

device.MacAddress = “00:11:22:AA:BB:CC"

We also need to specify the UUID that is being used by the remote device. (UUID : it's used to uniquely identify your application's Bluetooth service).

device.UUID = “f4144060-a8d0-11e5-b3ba-0002a5d5c51b”

The default UUID (“"00001101-0000-1000-8000-00805F9B34FB") is the one being used by most of the electronic peripherals, so you don’t need to provide it if the targeted device is HC-05.

Now connect to the device

device.connect()

TechTweaking.Bluetooth.BluetoothDevice::connect() will start a process to search for a device that matches its TechTweaking.Bluetooth.BluetoothDevice::Name or TechTweaking.Bluetooth.BluetoothDevice::MacAddress property. . If the device isn’t paired, the process will try to find it through a heavy and time consuming manner, so it's better to use pre-paired devices.

Connecting using Bluetooth devices list

The following two methods generate list of TechTweaking.Bluetooth.BluetoothDevice instances : TechTweaking.Bluetooth.Bt::showDevices(), TechTweaking.Bluetooth.Bt::getPairedDevices().

TechTweaking.Bluetooth.Bt::showDevices()

Display the default Android Bluetooth devices list. Subscribe to TechTweaking.Bluetooth.Bt::OnDevicePicked Event in order to get a reference of the picked device.

//The following code tries to connect to any device picked by user.
...
Bt.OnDevicePicked += HandleOnDevicePicked;
...
//Called when device is Picked by user
void HandleOnDevicePicked (BluetoothDevice device)
{
device.UUID = MY_UUID;
device.connect();
}

TechTweaking.Bluetooth.Bt::getPairedDevices()

returns a BluetoothDevice[] array of the paired devices on your Android.

Remarks
{ Get the Name and Mac Address using TechTweaking.Bluetooth.BluetoothDevice::Name and TechTweaking.Bluetooth.BluetoothDevice::MacAddress }

Connecting as a server

You can make a device act as a server that listen for incoming connection requests, by calling one of those methods : TechTweaking.Bluetooth.Bt::startServer(string), TechTweaking.Bluetooth.Bt::startServer(string,int), TechTweaking.Bluetooth.Bt::startServer(string,int,bool).

Bt.OnClientRequest += HandleOnClientRequest;
...
Bt.startServer(MY_UUID);
...
void HandleOnClientRequest (BluetoothDevice device)
{
device.connect();//< will connect to any Bluetooth device tries to initiate a connection with a UUID=MY_UUID.
}

Reading Data from a remote device

lets create an IEnumerator that will be started when a data receiving channel has just been instantiated.

IEnumerator ManageConnection (BluetoothDevice device)
{
while (device.IsReading) {//Reading loop
if (device.IsDataAvailable) {//check if there's data to read
byte [] msg = device.read (10);//read up to 10 bytes or less
if (msg.Length > 0 ) {
Debug.Log (System.Text.ASCIIEncoding.ASCII.GetString (msg));//convert bytes to string
}
}
yield return null;
}
}

The easiest way to start this IEnumerator is by assigning it to the TechTweaking.Bluetooth.BluetoothDevice::ReadingCoroutine property.

device.ReadingCoroutine = ManageConnection;

Also You can use the Event TechTweaking.Bluetooth.Bt::OnReadingStarted.

Optemizing the underlying threads/processes using the property TechTweaking.Bluetooth.BluetoothDevice::ThreadID

The library aims to be asynchronous and non-blocking, so no method call can affect your game loop flow. There are three operations that do extensive work without blocking which are sending data, calling a connect method and reading data by keep listening to the remote device.

Sending data is optemized and won't block, see TechTweaking.Bluetooth.BluetoothDevice::send().

Connecting also optemized and won't block, see TechTweaking.Bluetooth.BluetoothDevice::connect(), TechTweaking.Bluetooth.BluetoothDevice::connect(int).

On the other hand, reading data and listening to the remote device require a process/thread that will be running in the background, and if the device is listening to multiple remote devices at the same time, then having multiple threads/processes in the background is not always the most efficient way. You can make multiple devices share the same process/thread using the property TechTweaking.Bluetooth.BluetoothDevice::ThreadID by assigning the same non-zero int value to it for each device. Simply if TechTweaking.Bluetooth.BluetoothDevice::ThreadID equal to zero _ which is the default value_, the device will have its own single thread that isn't and won't be shared with others.
